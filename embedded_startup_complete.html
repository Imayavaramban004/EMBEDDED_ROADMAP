Understand the Two Concepts

Before writing a single line, you need to understand a distinction that confuses beginners and experts alike — and causes more bad firmware than any other single misunderstanding.

📄 Startup Code (narrow)
The Reset_Handler in startup.s or startup.c. Runs before main(). Sets up the C runtime — nothing else. Minimal. Automatic. Usually vendor-provided.
🏗 Startup Sequence (full)
Everything from Reset to steady-state operation: clock, watchdog, power, GPIO, peripherals, self-test, app init. Lives in main() and driver files. This is what you architect.
// The rule
Reset_Handler = minimal, vendor-provided, rarely touched. The startup sequence inside main() = your full responsibility, fully architected, the most consequential code you'll ever write. Both are "startup" — this guide covers both.
⚠ Common mistake
Thinking that because GPIO init "goes in main()" it's not part of startup. It IS. The startup sequence is everything that runs in main() before your application loop. Clock → watchdog → power → GPIO → peripherals → POST → app init. All of it is startup. All of it must be in the right order.

00
startup.s Reset_Handler — C Runtime Setup ▶
Copy .data Flash→RAM · Zero .bss · Call SystemInit() · Jump to main()

The Reset_Handler is the true firmware entry point — not main(). It runs before the C runtime exists. No global variables. No initialized data. No heap. Its only job: create conditions for C to run correctly.

⚠ Critical
Never call memset() or memcpy() here. The C library depends on .bss being zeroed before those functions work. Use raw pointer loops only.
  • ▸Copy .data — initialized globals live in Flash (LMA), but C expects them in RAM (VMA). Copy word-by-word.
  • ▸Zero .bss — uninitialized globals must be zero per the C standard. ROM contains garbage here.
  • ▸Call SystemInit() — vendor CMSIS: configures FPU, sets basic clock. Keep minimal.
  • ▸Jump to main() — never returns. Add while(1){} as a safety net.
startup.c — Reset_Handler
extern uint32_t _sidata, _sdata, _edata, _sbss, _ebss;

void Reset_Handler(void) {
    uint32_t *src = &_sidata, *dst = &_sdata;

    /* 1. Copy .data: Flash → RAM (NO memcpy!) */
    while (dst < &_edata) *dst++ = *src++;

    /* 2. Zero .bss (NO memset!) */
    for (dst = &_sbss; dst < &_ebss; ) *dst++ = 0U;

    /* 3. Vendor init — FPU, basic clock */
    SystemInit();

    /* 4. Jump to main — never returns */
    main();
    while(1) {}   /* safety net */
}
01
Layer 1 — Clock Clock Initialization ▶
HSI boot → HSE enable → PLL configure → switch SYSCLK → set bus dividers

Everything in your firmware runs at a frequency determined here. UART baud rates, ADC sample rates, PWM periods, RTOS tick — all derived from SYSCLK. Wrong clocks = subtle, non-reproducible bugs.

⚠ Most Missed Rule
Set Flash wait states BEFORE increasing clock frequency. Running at 72 MHz with 0 wait states causes random instruction fetch failures — one of the hardest bugs to diagnose.
SourceAccuracyStartupBest For
HSI / IRC~1–2%InstantBoot default, failsafe fallback
HSE / CrystalPPM level~msUART, USB, audio, precision
PLL (from HSE)PPM levelSlowestProduction — prefer this
LSI~5–10%FastIWDG watchdog
LSE / 32kHzPPM levelSlowRTC accurate timekeeping
  • ▸Flash wait states first (e.g. 2WS for 72 MHz on STM32F1) — BEFORE raising clock
  • ▸Enable HSE, wait ready — with timeout
  • ▸Configure PLL multiplier/prediv, enable, wait lock — with timeout
  • ▸Set AHB/APB1/APB2 bus dividers within rated limits
  • ▸Switch SYSCLK to PLL, confirm switch — with timeout
  • ▸Call SystemCoreClock_Update() — keeps HAL/CMSIS in sync
  • ▸If HSE fails → fall back to HSI, log error, return error status — never hang
clock.c — generalized pattern
Clock_Status_t Clock_Init(void) {
    uint32_t t;
    FLASH_SetWaitStates(2);              /* FIRST — before raising freq */

    HSE_Enable();
    t = Tick_ms();
    while (!HSE_Ready()) {
        if (Tick_ms() - t > 100) {
            Error_Log(ERR_HSE_TIMEOUT);
            return CLOCK_HSE_TIMEOUT;      /* never hang */
        }
    }
    PLL_Configure(PLL_SRC_HSE, 9, 1);    /* 8MHz × 9 = 72MHz */
    PLL_Enable();
    t = Tick_ms();
    while (!PLL_Locked()) {
        if (Tick_ms() - t > 100)
            return CLOCK_PLL_TIMEOUT;
    }
    BusDividers_Set(AHB_DIV1, APB1_DIV2, APB2_DIV1);
    SYSCLK_SetSource(PLL);
    SystemCoreClock_Update();
    return CLOCK_OK;
}
02
Layer 2 — Safety Watchdog Initialization ▶
Start IWDG right after clock is stable — before any peripheral init — non-negotiable

The watchdog resets the MCU if software stops refreshing it within the timeout. Start it early — right after clocks — so any subsequent init hang is automatically recovered. Log watchdog resets to NVM on every boot so field failures are diagnosable.

IWDG — Independent
Clocked from LSI (~40 kHz). Runs even if main clock fails. Cannot be stopped once started. Use for system-level protection.
WWDG — Window
Clocked from PCLK. Refresh too early OR too late triggers reset. Use to validate that your loop timing is correct.
  • ▸Check WDG reset flag first — log to NVM before anything else overwrites it
  • ▸Configure prescaler + reload for desired timeout (e.g., 1 second)
  • ▸Start IWDG — cannot be stopped after this point
  • ▸Call Watchdog_Refresh() first thing in every main loop iteration
  • ▸In safe mode — do NOT refresh. Let WDG reset the system automatically.
watchdog.c
/* LSI=40kHz, Prescaler=64 → 625 Hz tick → Reload=625 → 1s timeout */
void Watchdog_Init(void) {
    IWDG_Unlock();                     /* Write 0x5555 */
    IWDG_SetPrescaler(IWDG_PR_64);
    IWDG_SetReload(625);
    IWDG_Refresh();                    /* Write 0xAAAA */
    IWDG_Start();                      /* Write 0xCCCC — CANNOT STOP */
}
inline void Watchdog_Refresh(void) { IWDG_KR = 0xAAAA; }
bool        Watchdog_WasReset(void) { return RCC->CSR & RCC_CSR_IWDGRSTF; }
03
Layer 3 — Power Power & Voltage Initialization ▶
Voltage scale · Brown-Out Reset · Vref enable · Supply rail validation

An MCU operating below its rated supply voltage causes Flash write corruption, wrong ADC readings, and random resets. Configure BOR to auto-reset the MCU if VDD drops below safe threshold, and measure VDD to confirm the supply is healthy before continuing.

  • ▸Enable PWR peripheral clock
  • ▸Set voltage scale matching target SYSCLK (higher freq = higher scale)
  • ▸Configure BOR level — e.g. BOR Level 3 → reset below ~2.7 V
  • ▸Enable internal Vref — required for accurate ADC readings
  • ▸Measure VDD via ADC and validate min/max limits
  • ▸If undervolt detected → safe mode. Never operate with a starved supply.
04
Layer 4 — GPIO GPIO Initialization ▶
All pins → Analog first (safe default) · Then configure only used pins · All assignments in board_config.h

GPIO init has two jobs: (1) put ALL unused pins in a safe, low-power state — Analog Input — to prevent floating inputs and minimize EMI. (2) configure functional pins. Define every pin assignment in one place: board_config.h.

// Industry Rule
Set ALL GPIO pins to Analog Input mode first — lowest leakage, lowest EMI, no spurious interrupts. Then configure only the pins your design uses. Unconfigured floating pins are a common EMI compliance failure.
  • ▸Enable all GPIO port clocks in one shot
  • ▸Set ALL pins to Analog Input first — safe default
  • ▸Configure output pins (LEDs): Push-Pull, Low speed, drive to known OFF state
  • ▸Configure input pins (buttons): Input, Pull-Up or Pull-Down
  • ▸Configure peripheral alternate function pins: AF mode, correct speed
gpio.c — correct pattern
void GPIO_Init_All(void) {
    RCC_GPIO_ClockEnable(GPIO_CLK_ALL);
    GPIO_SetAllAnalog();           /* safe default — ALL pins */

    /* LEDs: output, push-pull, low speed, initially OFF */
    GPIO_InitTypeDef g = {GPIO_MODE_OUTPUT_PP, GPIO_NOPULL, GPIO_SPEED_LOW};
    g.Pin = LED_STATUS_PIN | LED_ERROR_PIN;
    GPIO_Init(GPIOA, &g);
    GPIO_Write(LED_STATUS_PORT, LED_STATUS_PIN, 0);

    /* UART1: alternate function, high speed */
    g.Mode = GPIO_MODE_AF_PP; g.Speed = GPIO_SPEED_HIGH;
    g.Alternate = GPIO_AF7_USART1;
    g.Pin = UART1_TX_PIN | UART1_RX_PIN;
    GPIO_Init(UART1_PORT, &g);
}
05
Layer 5 — Drivers Peripheral Initialization ▶
Dependency order · Every init returns status · NVIC priorities set here

Peripherals must be initialized in dependency order — a peripheral that depends on another must come after it. Every init function must return a status code. Handle every failure.

SysTick→ DMA→ Debug UART→ SPI / I2C→ ADC + Cal→ Timers / PWM
  • ▸SysTick first — all delay/timeout functions depend on it
  • ▸DMA before peripherals — SPI, UART Rx, ADC may all use DMA
  • ▸Debug UART early — so all subsequent failures can be logged to serial
  • ▸ADC self-calibration — must run right after ADC init, before any reading
  • ▸NVIC priorities — configure all interrupt priorities before enabling any IRQ
  • ▸Non-critical failure (debug UART) → log warning, continue
  • ▸Critical failure (SysTick, DMA) → safe mode immediately
peripherals.c — orchestrator
Init_Status_t Peripherals_Init(void) {
    /* SysTick — critical; abort if fails */
    if (SysTick_Init(1000) != INIT_OK) return INIT_FAIL;
    DMA_Init();
    /* UART — non-critical (warn only) */
    if (UART_Init(DEBUG_UART, 115200) != INIT_OK)
        Error_Log(ERR_UART_WARN);
    SPI_Init(SPI1, MODE_MASTER, 1000000);
    I2C_Init(I2C1, 400000);
    ADC_Init();
    ADC_SelfCalibrate();       /* MANDATORY after ADC init */
    NVIC_SetPriority(DMA_IRQn,   0);   /* highest */
    NVIC_SetPriority(UART_IRQn,  1);
    NVIC_SetPriority(SysTick_IRQn, 15); /* lowest */
    return INIT_OK;
}
06
Layer 6 — Self-Test Power-On Self-Test (POST) ▶
RAM test · Flash CRC · Sensor range · Voltage rails · Fail fast, fail visibly

POST validates hardware integrity before entering normal operation. A system that silently operates with corrupted RAM is worse than one that fails visibly at boot. Required by IEC 61508 (industrial) and ISO 26262 (automotive).

  • ▸RAM March-C test — write 0xAA/0x55/0xFF/0x00 to each location, read back. Detects stuck bits.
  • ▸Flash CRC32 — compute CRC over firmware image, compare against reference stored at build time
  • ▸Sensor plausibility — read each sensor, verify physically possible range (e.g. temp −40°C to +125°C)
  • ▸Voltage rail check — measure all rails via ADC, verify within ±5% of nominal
  • ▸Refresh watchdog between each test — POST can be slow
  • ▸Any failure → log bitmask to NVM + safe mode. Never silently ignore POST failure.
self_test.c
#define POST_FAIL_RAM    0x01u
#define POST_FAIL_FLASH  0x02u
#define POST_FAIL_SENSOR 0x04u
#define POST_FAIL_VOLT   0x08u

uint32_t POST_RunAll(void) {
    uint32_t result = 0;
    Watchdog_Refresh();
    if (!RAM_MarchC_Test(&_heap_start, &_heap_end)) result |= POST_FAIL_RAM;
    Watchdog_Refresh();
    if (CRC32_Calc(FLASH_BASE, FLASH_SIZE) != STORED_CRC) result |= POST_FAIL_FLASH;
    float t = Sensor_ReadTemp();
    if (t < -40.0f || t > 125.0f)  result |= POST_FAIL_SENSOR;
    if (result) System_EnterSafeMode(result);  /* log + WDG reset */
    return result;
}
07
Layer 7 — Application Application Initialization ▶
NVM config load · Calibrate sensors · Init state machines · Create RTOS tasks

Hardware is verified and running. Application init is completely free of direct register access — all hardware interaction goes through driver APIs. Load config, validate it, apply calibration, init state machines, create RTOS objects.

  • ▸Read config from NVM/EEPROM — if CRC invalid, load defaults and write them back
  • ▸Validate every config field — bounds-check before applying. Corrupted config is common in field.
  • ▸Apply calibration — load sensor gain/offset from NVM, pass to driver APIs
  • ▸Init state machines — set initial state, clear history
  • ▸RTOS tasks, queues, semaphores — create ALL before calling vTaskStartScheduler()
  • ▸Signal successful startup — 3× LED blink, print debug banner with FW version
08
Running State Main Loop / RTOS Scheduler ▶
System fully initialized · WDG refresh every iteration · Never returns from main()

The system is fully initialized and verified. Enter the superloop or start the RTOS scheduler. Watchdog must be refreshed every iteration. main() must never return.

  • ▸Watchdog_Refresh() is the first call in every loop iteration
  • ▸Use WFI (Power_EnterSleep()) at the bottom — wake on interrupt for power efficiency
  • ▸For RTOS: vTaskStartScheduler() — never returns, replaces while(1)
  • ▸If somehow exiting main() — hit while(1){} safety net. Never fall off end of main.
main.c — loop options
/* Option A: Bare-metal superloop */
while (1) {
    Watchdog_Refresh();        /* FIRST — every iteration */
    App_RunCycle();             /* state machine tick */
    Power_EnterSleep();         /* WFI — wake on interrupt */
}

/* Option B: FreeRTOS */
vTaskStartScheduler();          /* never returns */

/* Safety net — should NEVER execute */
while (1) {}                    /* WDG resets if reached */

The Complete main() — Copy-Paste Ready

main() is a sequencer — not a place for register writes or magic numbers. One function call per layer, return value checked on every call.

main.c — Production Embedded Template
#include "clock.h"
#include "watchdog.h"
#include "power.h"
#include "gpio.h"
#include "peripherals.h"
#include "self_test.h"
#include "app.h"

#define FW_MAJOR  1U
#define FW_MINOR  0U
#define FW_PATCH  0U

int main(void)
{
    /* ── Interrupt priority grouping — must be first ─────────────── */
    NVIC_SetPriorityGrouping(NVIC_PRIORITYGROUP_4);

    /* ── Layer 1: Clock — everything depends on this ─────────────── */
    if (Clock_Init() != CLOCK_OK)
        Error_BlinkForever(ERR_CLOCK);       /* can't log yet */

    /* ── Layer 2: Watchdog — start BEFORE any peripheral init ─────── */
    if (Watchdog_WasReset()) NVM_Log(LOG_WDG_RESET);
    Watchdog_Init();                         /* starts — cannot stop */

    /* ── Layer 3: Power — voltage validation ─────────────────────── */
    if (Power_Init() != PWR_OK)
        System_EnterSafeMode(ERR_UNDERVOLT);

    /* ── Layer 4: GPIO — safe state all pins first ───────────────── */
    GPIO_Init_All();

    /* ── Layer 5: Peripherals — in dependency order ──────────────── */
    if (Peripherals_Init() == INIT_FAIL)
        System_EnterSafeMode(ERR_PERIPHERAL);

    /* UART is up — print debug banner */
    Debug_Printf("FW v%u.%u.%u | SYSCLK %lu Hz\r\n",
                  FW_MAJOR, FW_MINOR, FW_PATCH, SystemCoreClock);

    /* ── Layer 6: Self-Test (POST) ───────────────────────────────── */
    Watchdog_Refresh();
    if (POST_RunAll() != POST_OK)
        System_EnterSafeMode(ERR_POST);

    /* ── Layer 7: Application Init ───────────────────────────────── */
    Watchdog_Refresh();
    App_Init();

    /* ── Layer 8: Main Loop ──────────────────────────────────────── */
    while (1) {
        Watchdog_Refresh();              /* ALWAYS first */
        App_RunCycle();
        Power_EnterSleep();              /* WFI — power saving */
    }

    /* Should NEVER reach here */
    ASSERT_NEVER_REACH();
    return 0;
}

/* ── Safe Mode — log, blink error LED, let WDG reset ─────────── */
void System_EnterSafeMode(uint32_t err) {
    NVM_Log_Error(err);
    Debug_Printf("[SAFE MODE] 0x%08lX\r\n", err);
    while (1) {
        LED_Toggle(LED_ERROR);
        Delay_ms(100);
        /* Do NOT refresh WDG here — let it reset the system */
    }
}

Two Critical Patterns — Memorize These

✗ Wrong — Infinite poll
while (!HSE_Ready()) {} // ← if crystal fails → bricked forever
✓ Right — Always timeout
if (Tick()-t > 100) return TIMEOUT; // ← safe
✗ Wrong — void init
void ADC_Init(void) {...} // ← rejected in code review
✓ Right — return status
Init_Status_t ADC_Init(void) {...} // ← production

Final Checklist — Before You Ship

  • 01Reset_Handler copies .data and zeros .bss using raw pointer loops — not memcpy / memset
  • 02Flash wait states set before increasing clock frequency
  • 03Every oscillator/PLL wait has a timeout with a meaningful error return path
  • 04Clock init falls back to HSI and logs an error if HSE fails — never hangs
  • 05Watchdog started immediately after clock is stable — before GPIO or peripheral init
  • 06Watchdog reset flag checked and logged to NVM on every boot
  • 07BOR (Brown-Out Reset) configured — MCU resets automatically if VDD drops
  • 08All GPIO pins set to Analog Input before configuring functional pins
  • 09All GPIO pin assignments defined in one file (board_config.h) — never scattered
  • 10Every init function returns a status code — no void init functions in production
  • 11Peripherals initialized in dependency order: SysTick → DMA → UART → SPI/I2C → ADC
  • 12ADC self-calibration called immediately after ADC_Init()
  • 13NVIC priorities configured before enabling any IRQ
  • 14POST runs RAM test, Flash CRC, sensor range, and voltage rails
  • 15NVM config read has CRC validity check — loads defaults if corrupted
  • 16Watchdog_Refresh() is the first call in every main loop iteration
  • 17System_EnterSafeMode() logs to NVM and does NOT refresh watchdog — lets WDG auto-reset
  • 18main() never returns — while(1) or vTaskStartScheduler() as final statement
  • 19Firmware version defined in one place — printed in debug banner after UART init
  • 20No register writes inside main.c — only driver API calls
