UART & USART
Bare-Metal Production Guide

Learn the exact steps to configure UART and USART on STM32F4 from scratch — no HAL library, no magic. Every register, every bit, explained simply.

🔶 UART (Async Only)

UART4 & UART5. Transmits and receives data without a shared clock. Both sides must agree on the same baud rate in advance. Simpler to wire — just TX and RX.

🔷 USART (Async + Sync)

USART1, 2, 3, 6. Has all UART features plus an optional CK pin for synchronous mode (like SPI). More flexible but slightly more complex to set up.

💡
Beginner tip — the simplest mental model:
Think of UART like two people talking on walkie-talkies. They agree beforehand on a speaking speed (baud rate). No shared clock signal needed.
USART is the same walkie-talkie but with an extra wire (CK) that can sync both sides to the same heartbeat — useful when you need zero timing error.
⚠️
Key rule — always enable the clock before touching registers.
On STM32F4, every peripheral is clock-gated by default. Writing to GPIO or UART registers without enabling their clock first produces no effect and is a very common beginner mistake.
✅
Production rule — set UE (UART Enable) last.
Configure BRR, CR2, CR3, and CR1 fields (word length, parity, TE, RE) while UE=0. Then set UE=1 as the final step. This prevents partial configuration from reaching the line.
UART

UART4 Step-by-Step Flow

UART4 is a pure asynchronous peripheral on the APB1 bus. It does NOT have a CK pin. The example below uses PA0 (TX) and PA1 (RX) at 115200 baud, HSI 16 MHz.

1
Enable Peripheral Clocks (RCC)
Why? The STM32F4 powers off all peripherals by default to save energy. You must tell the RCC (Reset & Clock Control) to turn on the clocks for GPIOA (for the pins) and UART4 (for the peripheral) before touching any of their registers.
RCC_AHB1ENR 0x40023830  (offset 0x30)
Bit 0 = 1  // GPIOAEN — enable GPIOA clock
Reset value: 0x0000 0000
RCC_APB1ENR 0x40023840  (offset 0x40)
Bit 19 = 1  // UART4EN — enable UART4 clock
Reset value: 0x0000 0000 
/* Always read back after enabling a clock — flushes the APB pipeline */
RCC_AHB1ENR |= (1UL << 0);   // GPIOAEN
(void)RCC_AHB1ENR;

RCC_APB1ENR |= (1UL << 19);  // UART4EN
(void)RCC_APB1ENR;
2
Configure GPIO Pins → Alternate Function Mode
Why? By default, all GPIO pins are plain digital inputs. To make PA0 become a UART TX line, you must point it to AF8 (Alternate Function 8 = UART4). The MCU will then route the UART4 transmitter to the physical pad automatically. Configure all registers before setting MODER to AF — this prevents glitches.
📌
PA0 and PA1 both use AF8 for UART4. AFRL covers pins 0–7 (4 bits each).
Correct order: OTYPER → OSPEEDR → PUPDR → AFRL → MODER last.
GPIOA RegistersBase: 0x4002 0000
MODER +0x00   PA0[1:0]=10, PA1[3:2]=10  // Alternate Function
OTYPER +0x04   bit0=0, bit1=0              // Push-Pull
OSPEEDR+0x08   PA0[1:0]=00, PA1[3:2]=00  // Low Speed
PUPDR +0x0C   PA0[1:0]=01, PA1[3:2]=01  // Pull-Up
AFRL +0x20   [3:0]=1000(AF8), [7:4]=1000(AF8) // UART4 
/* ---- PA0 TX ---- */
GPIOA_OTYPER  &= ~(1UL << 0);               // Push-Pull
GPIOA_OSPEEDR &= ~(3UL << 0);               // Low Speed
GPIOA_PUPDR   &= ~(3UL << 0); GPIOA_PUPDR |= (1UL << 0); // Pull-Up
GPIOA_AFRL    &= ~(0xFUL << 0); GPIOA_AFRL |= (8UL << 0); // AF8
GPIOA_MODER   &= ~(3UL << 0); GPIOA_MODER |= (2UL << 0); // AF mode last

/* ---- PA1 RX ---- */
GPIOA_OTYPER  &= ~(1UL << 1);
GPIOA_OSPEEDR &= ~(3UL << 2);
GPIOA_PUPDR   &= ~(3UL << 2); GPIOA_PUPDR |= (1UL << 2);
GPIOA_AFRL    &= ~(0xFUL << 4); GPIOA_AFRL |= (8UL << 4);
GPIOA_MODER   &= ~(3UL << 2); GPIOA_MODER |= (2UL << 2);
3
Configure the UART4 Peripheral Registers
Why? Now that the GPIO is set up, configure the UART4 engine itself: how fast to communicate (baud rate), how many data bits, how many stop bits, and whether to use parity checking. Always configure these before enabling UART (UE bit).
UART4 RegistersBase: 0x4000 4C00
SR +0x00  // Status — read to check TXE, RXNE, errors
DR +0x04  // Data — write=TX byte, read=RX byte
BRR +0x08  = 138     // 16 000 000 / 115200
CR1 +0x0C  : M=0(8bit), PCE=0, TE=1, RE=1, UE=1(last)
CR2 +0x10  : STOP[13:12]=00  // 1 stop bit
CR3 +0x14  : RTSE=0, CTSE=0   // no hardware flow control 
/* Step 3a — disable before configuration */
UART4_CR1 &= ~(1UL << 13);   // UE=0

/* Step 3b — baud rate */
UART4_BRR  = 138U;            // 16 000 000 / 115200 ≈ 138

/* Step 3c — 1 stop bit (CR2 STOP field = 00) */
UART4_CR2 &= ~(3UL << 12);

/* Step 3d — no hardware flow control */
UART4_CR3 &= ~((1UL << 8) | (1UL << 9));

/* Step 3e — 8N1: M=0 (8bit), PCE=0 (no parity), TE+RE enabled */
uint32_t cr1 = 0U;
cr1 |= (1UL << 3);            // TE=1
cr1 |= (1UL << 2);            // RE=1
UART4_CR1 = cr1;

/* Step 3f — enable UART last*/
UART4_CR1 |= (1UL << 13);    // UE=1
4
Initialise SysTick for Millisecond Delays
Why? Your application probably needs to wait (delay) between transmissions. SysTick is a free built-in timer inside the Cortex-M4 core. You set it to tick every 1 ms, and a simple counter gives you delay_ms() without needing an external timer peripheral.
SysTick RegistersCore PPB — PM0214 §4.4
STK_CTRL 0xE000E010  ← 0x0 then 0x7
STK_LOAD 0xE000E014  ← 15999  // 16MHz/1000-1
STK_VAL 0xE000E018  ← 0       // clear counter 
STK_CTRL = 0;                         // disable first
STK_LOAD = (16000000UL / 1000U) - 1U; // 15999
STK_VAL  = 0;                         // clear current value
STK_CTRL = 0x7;  // CLKSOURCE=1 | TICKINT=1 | ENABLE=1
5
Transmit and Receive Data
How to send a byte: Wait until SR.TXE=1 (transmit register empty), then write your byte to DR.
How to receive a byte: Wait until SR.RXNE=1 (received data ready), read DR, then check SR for errors. 
/* --- Transmit one byte --- */
void UART4_SendByte(uint8_t byte) {
    while (!(UART4_SR & (1<<7)));  // wait TXE=1
    UART4_DR = byte;
}

/* --- Receive one byte --- */
uint8_t UART4_ReadByte(void) {
    while (!(UART4_SR & (1<<5)));  // wait RXNE=1
    return (uint8_t)UART4_DR;
}

/* --- Main loop --- */
UART4_SendByte('H'); UART4_SendByte('i'); UART4_SendByte('\n');
SysTick_DelayMs(1000);
USART

USART1 Step-by-Step Flow

USART1 sits on the APB2 bus (faster, up to 84 MHz). It uses PA9 (TX) and PA10 (RX) with AF7. The steps are identical to UART except for clock registers and AF number. Optional CK pin (PA8) is covered in Step 3b.

1
Enable Peripheral Clocks (RCC)
Key difference from UART: USART1 is on APB2 (not APB1). Its enable bit is in RCC_APB2ENR, bit 4. The GPIOA clock comes from AHB1 as before.
RCC_AHB1ENR0x40023830
Bit 0 = 1  // GPIOAEN (same as UART)
RCC_APB2ENR0x40023844  (offset 0x44)
Bit 4 = 1  // USART1EN — different register & bit from UART4!
RCC_AHB1ENR |= (1UL << 0);  // GPIOAEN
(void)RCC_AHB1ENR;

RCC_APB2ENR |= (1UL << 4);  // USART1EN — APB2, bit 4
(void)RCC_APB2ENR;
2
Configure GPIO Pins → AF7 (USART1)
Key difference: USART1 uses AF7 (not AF8). TX is PA9, RX is PA10. Both need high speed because USART1 runs at up to 84 MHz on APB2. If you use synchronous mode, also configure PA8 as CK output with AF7.
GPIOA RegistersBase: 0x4002 0000
MODER   PA9[19:18]=10, PA10[21:20]=10  // AF mode
OTYPER   bit9=0, bit10=0                // Push-Pull
OSPEEDR  PA9[19:18]=10, PA10[21:20]=10  // High Speed
PUPDR   PA9[19:18]=00, PA10[21:20]=01  // TX:None, RX:PU
AFRH   PA9[7:4]=0111(AF7), PA10[11:8]=0111(AF7) 
/* PA9 = TX, PA10 = RX — both AF7 — AFRH (pins 8-15) */

/* PA9 TX */
GPIOA_OTYPER  &= ~(1UL << 9);
GPIOA_OSPEEDR &= ~(3UL << 18); GPIOA_OSPEEDR |= (2UL << 18); // High
GPIOA_PUPDR   &= ~(3UL << 18);                                   // No pull
GPIOA_AFRH    &= ~(0xFUL << 4); GPIOA_AFRH |= (7UL << 4);      // AF7
GPIOA_MODER   &= ~(3UL << 18); GPIOA_MODER |= (2UL << 18);    // AF last

/* PA10 RX */
GPIOA_OTYPER  &= ~(1UL << 10);
GPIOA_OSPEEDR &= ~(3UL << 20); GPIOA_OSPEEDR |= (2UL << 20);
GPIOA_PUPDR   &= ~(3UL << 20); GPIOA_PUPDR   |= (1UL << 20); // PU on RX
GPIOA_AFRH    &= ~(0xFUL << 8); GPIOA_AFRH |= (7UL << 8);
GPIOA_MODER   &= ~(3UL << 20); GPIOA_MODER |= (2UL << 20);
3
Configure the USART1 Peripheral Registers
Key difference: USART1 clock from APB2 = 84 MHz (when PLL is running). At 16 MHz HSI (default), APB2 = 16 MHz — same as APB1 for UART4. USART also has CR2.CLKEN (synchronous CK enable), CR2.CPHA, CR2.CPOL — not available on pure UART.
USART1 RegistersBase: 0x4001 1000
SR +0x00  // Status: TXE[7], RXNE[5], TC[6], ORE[3]
DR +0x04  // Data register
BRR +0x08  = 138  // 16 MHz HSI / 115200
CR1 +0x0C  : OVER8=0(16x), M=0, PCE=0, TE=1, RE=1, UE=1
CR2 +0x10  : STOP=00, CLKEN=0  // async mode (CLKEN=0)
CR3 +0x14  : RTSE=0, CTSE=0
✅
USART-only feature — Synchronous Mode (CR2.CLKEN=1):
Set CLKEN=1 to output a clock on PA8 (CK pin). Useful for clocking a slave device that needs a master clock. UART4/5 cannot do this.
USART1_CR1 &= ~(1UL << 13);  // UE=0 — disable first
USART1_BRR  = 138U;           // 16 000 000 / 115200
USART1_CR2 &= ~(3UL << 12); // 1 stop bit (STOP=00)
USART1_CR2 &= ~(1UL << 11); // CLKEN=0 (async mode)
USART1_CR3 &= ~((1<<8)|(1<<9));  // no flow control

uint32_t cr1 = (1UL<<3) | (1UL<<2);  // TE=1 RE=1
USART1_CR1 = cr1;
USART1_CR1 |= (1UL << 13);  // UE=1 — enable last
4
SysTick Init & Transmit / Receive
Identical to UART. Same SysTick setup. Same polling loop on SR.TXE / SR.RXNE. Just change the register names. 
/* Transmit */
while (!(USART1_SR & (1<<7)));  // TXE=1
USART1_DR = 'A';

/* Receive */
while (!(USART1_SR & (1<<5)));  // RXNE=1
uint8_t b = (uint8_t)USART1_DR;

/* Delay */
SysTick_DelayMs(1000);

Cheat Sheet — Common Operations

🔶 UART4 — BRR Formula
// BRR = fCK / baud  (OVER8=0, 16x oversampling)
// fCK = APB1 clock frequency

115200 baud @ 16 MHz : BRR = 138
9600   baud @ 16 MHz : BRR = 1666
115200 baud @ 42 MHz : BRR = 364
115200 baud @ 84 MHz : BRR = 729
🔷 USART1 — BRR Formula
// APB2 clock (84 MHz with PLL)

115200 baud @ 16 MHz : BRR = 138
115200 baud @ 84 MHz : BRR = 729
1 Mbaud    @ 84 MHz : BRR = 84
9600 baud  @ 84 MHz : BRR = 8750
SR Flag Bits (both UART & USART)
SR bit 7  = TXE   TX register empty
SR bit 6  = TC    Transmission complete
SR bit 5  = RXNE  RX data ready to read
SR bit 3  = ORE   Overrun error
SR bit 1  = FE    Framing error
SR bit 0  = PE    Parity error
CR1 Key Bits
bit 15 = OVER8  0=16x oversampling (default)
bit 13 = UE     UART Enable (set LAST)
bit 12 = M      Word: 0=8bit, 1=9bit
bit 10 = PCE    Parity enable
bit  9 = PS     0=even, 1=odd parity
bit  3 = TE     Transmitter enable
bit  2 = RE     Receiver enable
CR2 Key Bits (USART extra)
bits[13:12] = STOP   00=1bit, 10=2bits
bit 11      = CLKEN  CK pin enable (USART only)
bit 10      = CPOL   Clock polarity (sync)
bit  9      = CPHA   Clock phase (sync)
bit  8      = LBCL   Last bit clock (sync)
CR3 Key Bits
bit 11 = ONEBIT  1-sample bit method
bit  9 = CTSE    CTS flow control
bit  8 = RTSE    RTS flow control
bit  7 = DMAT    DMA transmit enable
bit  6 = DMAR    DMA receive enable
bit  3 = HDSEL   Half-duplex mode
bit  1 = IREN    IrDA mode enable
