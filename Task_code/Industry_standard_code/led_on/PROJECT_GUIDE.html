2Project File Structure

Every file has exactly one responsibility. Adding UART later only touches the UART driver, not startup or main.

📋 inc/stm32f4xx_gpio.h

The contract layer. Contains base addresses, all RCC offsets (0x00–0x94), GPIO offsets, REG32 macro, bit-field constants, GPIO_Config_t struct, and public API prototypes. No implementation code.

⚙️ src/stm32f4xx_gpio.c

Implements the API: validate → clock → BSRR → OTYPER → OSPEEDR → PUPDR → MODER. Internal helpers are static — invisible outside this file.

🚀 startup/startupstm32f4.c

First code that runs on reset. Provides the Cortex-M4 vector table, copies .data from flash, zeros .bss, calls SystemInit(), then calls main().

📜 linker/stm32f4.ld

Tells ld where to place every section in 512K flash / 128K SRAM. Defines _estack, _sdata, _etext symbols that startup code reads.

Why two addresses for .data?

Initial values live in flash (LMA). CPU accesses them in SRAM at runtime (VMA). Reset_Handler copies LMA→VMA before main() runs.

⚠ Always use BSRR, never ODR for individual pin control

ODR requires read-modify-write — non-atomic, unsafe in ISR. BSRR is write-only: bits written 1 take effect, bits written 0 are ignored. One 32-bit store = atomic set+reset.

Ctypedef enum {
    GPIO_OK               =  0,
    GPIO_ERR_NULL_PTR     = -1,
    GPIO_ERR_INVALID_PIN  = -2,
    GPIO_ERR_INVALID_MODE = -3
} GPIO_Status_t;

typedef struct {
    volatile uint32_t *port_base; /* e.g. (volatile uint32_t*)GPIOA_BASE */
    uint32_t   rcc_bit;           /* RCC_AHB1ENR bit for this port        */
    uint8_t    pin;               /* 0–15                                 */
    uint8_t    mode;              /* GPIO_MODER_*                         */
    uint8_t    otype;             /* GPIO_OTYPER_*                        */
    uint8_t    speed;             /* GPIO_OSPEEDR_*                       */
    uint8_t    pull;              /* GPIO_PUPDR_*                         */
    uint8_t    init_state;        /* GPIO_PIN_SET / GPIO_PIN_RESET        */
} GPIO_Config_t;

GPIO_Status_t GPIO_Init(const GPIO_Config_t *cfg);
GPIO_Status_t GPIO_WritePin(const GPIO_Config_t *cfg, GPIO_PinState_t state);
GPIO_Status_t GPIO_ReadPin(const GPIO_Config_t *cfg, GPIO_PinState_t *state);
GPIO_Status_t GPIO_TogglePin(const GPIO_Config_t *cfg);
GPIO_Status_t GPIO_DeInit(const GPIO_Config_t *cfg);

C/*
 * REG32(Base, Offset)  →  *(volatile uint32_t*)(Base + Offset)
 * volatile ensures every read/write is a real bus transaction.
 */
#define REG32(Base, Offset) \
    (*(volatile uint32_t *)((Base) + (Offset)))

/* LED blinking on GPIOA */
#define RCC_AHB1ENR   REG32(RCC_BASE,   RCC_AHB1ENR_OFFSET)
#define GPIOA_MODER   REG32(GPIOA_BASE, GPIO_MODER_OFFSET)
#define GPIOA_BSRR    REG32(GPIOA_BASE, GPIO_BSRR_OFFSET)
Why volatile?

Without volatile the compiler may cache register values in a CPU register and skip real memory accesses. With it, every access generates a real bus transaction to the exact hardware address.

C · Reset_Handler/* ① .data copy */
uint32_t data_size = (uint32_t)&_edata - (uint32_t)&_sdata;
uint8_t *src  = (uint8_t *)&_etext;
uint8_t *dest = (uint8_t *)&_sdata;
for (uint32_t i = 0; i < data_size; i++) { dest[i] = src[i]; }

/* ② .bss zero */
uint32_t bss_size = (uint32_t)&_ebss - (uint32_t)&_sbss;
uint8_t *bss = (uint8_t *)&_sbss;
for (uint32_t i = 0; i < bss_size; i++) { bss[i] = 0; }

SystemInit();  (void)main();  Default_Handler();
WEAK_HANDLER macro — silence -Wmissing-attributes

All 97 IRQ aliases point to Default_Handler which is noreturn. Every alias must also carry noreturn. The macro WEAK_HANDLER(name) bundles weak + alias + noreturn — the CMSIS standard approach.

Section 07
7Linker Script — Section Layout

The linker script maps every section to an address. Without it, code lands wherever ld chooses — wrong for embedded.

📀 FLASH — 0x0800_0000 (512K)
.isr_vector
Interrupt Vector Table

First 4 bytes = _estack. Next = Reset_Handler. KEEP() prevents --gc-sections removing it.

.text
Executable code

All compiled functions. _etext = LMA start of .data values.

.rodata
Read-only data

String literals, const globals (e.g. GPIO_Config_t LED_PA5).

.data (LMA)
Init values in flash

Reset_Handler copies these to SRAM VMA. AT > FLASH directive.

🧠 SRAM — 0x2000_0000 (128K)
.data (VMA)
Initialised globals

Copied here by Reset_Handler. _sdata to _edata.

.bss
Zero-initialised globals

No flash storage needed. Zeroed by Reset_Handler. C standard requirement.

.noinit
Survives soft reset

Neither copied nor zeroed. Use for fault logs and watchdog counters.

Stack ↓
Grows from _estack downward

_estack = top of SRAM. Min size checked by ASSERT at link time.

Linker ScriptENTRY(Reset_Handler)

MEMORY {
  FLASH (RX)  : ORIGIN = 0x08000000, LENGTH = 512K
  SRAM  (RWX) : ORIGIN = 0x20000000, LENGTH = 128K
}

PROVIDE(_estack = ORIGIN(SRAM) + LENGTH(SRAM)); /* top of SRAM → initial SP */

.data : {
  _sdata = .;   /* VMA — CPU accesses here at runtime (SRAM) */
  *(.data*);
  _edata = .;
} > SRAM AT > FLASH  /* AT > FLASH — initial values stored in flash */

_lma_data_start = LOADADDR(.data);  /* = _etext: LMA for Reset_Handler */
Section 08
8GPIO Driver — Exact Register Write Sequence

The order of writes matters. MODER before BSRR causes a pin glitch. Reference: RM0090 §8.4

✅
gpio_validate_config(cfg)
ptr ≠ NULL, pin ≤ 15, mode/speed/pull in valid range
⬇ RM0090 §7.3.10
① RCC
RCC_AHB1ENR |= rcc_bit
Enable GPIOA clock. NEVER access GPIO registers before this!
⬇ RM0090 §8.4.7
② BSRR
Pre-set output state BEFORE MODER
Atomic write. Avoids glitch when MODER transitions input→output
⬇ RM0090 §8.4.2
③ OTYPER
Push-pull (0) or open-drain (1) — 1 bit/pin
Push-pull for LED: drives both HIGH and LOW actively
⬇ RM0090 §8.4.3
④ OSPEEDR
Speed: LOW / MEDIUM / HIGH / VERY HIGH — 2 bits/pin
LOW for LED: less EMI, lower power consumption
⬇ RM0090 §8.4.4
⑤ PUPDR
NONE / PULL-UP / PULL-DOWN — 2 bits/pin
NONE for output — MCU drives the pin, no pull needed
⬇ RM0090 §8.4.1 — LAST!
⑥ MODER
Mode = 01 (output) — written LAST
Pin switches from input→output only when MODER is written
⬇
✅
return GPIO_OK
PA5 configured. LED at init_state. Call GPIO_WritePin now.
🚨 Never access a GPIO register before enabling its RCC clock

Reading or writing a peripheral with no clock on Cortex-M4 causes a BusFault → HardFault. Always OR-in RCC_AHB1ENR bit first, then access GPIOx registers.

Section 09
9main.c — Application Layer

main.c never touches a register directly — it only calls the GPIO API. The config is const so the linker stores it in flash (zero RAM cost).

C · main.c#include "stm32f4xx_gpio.h"

static const GPIO_Config_t LED_PA5 = {
    .port_base  = (volatile uint32_t *)GPIOA_BASE,
    .rcc_bit    = RCC_AHB1ENR_GPIOAEN,  /* bit 0 in RCC_AHB1ENR */
    .pin        = 5U,                    /* PA5 = Nucleo onboard LED */
    .mode       = GPIO_MODER_OUTPUT,
    .otype      = GPIO_OTYPER_PP,
    .speed      = GPIO_OSPEEDR_LOW,
    .pull       = GPIO_PUPDR_NONE,
    .init_state = GPIO_PIN_RESET,
};

int main(void) {
    GPIO_Status_t status;
    status = GPIO_Init(&LED_PA5);
    if (status != GPIO_OK) Error_Handler(status);

    status = GPIO_WritePin(&LED_PA5, GPIO_PIN_SET);  /* LED ON */
    if (status != GPIO_OK) Error_Handler(status);

    for (;;) { }
}

✅ Why const config struct?

The compiler places const objects in .rodata → flash. Zero SRAM cost. For 10 GPIO pins: 10×12 = 120 bytes saved from SRAM.

✅ Why Error_Handler not assert()?

assert() pulls in heavyweight libc string formatting. A bare-metal while(1) trap uses 4 bytes and lets the watchdog reset — production safe.

# Using one combined SRCS with a single patsubst leaves startup/*.c
# unchanged in OBJS, which ends up in DEPS, causing make to -include
# the .c file as a Makefile → "missing separator" error.
SRC_SRCS     := $(wildcard $(SRC_DIR)/*.c)
STARTUP_SRCS := $(wildcard $(STARTUP_DIR)/*.c)
SRC_OBJS     := $(patsubst $(SRC_DIR)/%.c,     $(BUILD_DIR)/%.o, $(SRC_SRCS))
STARTUP_OBJS := $(patsubst $(STARTUP_DIR)/%.c, $(BUILD_DIR)/%.o, $(STARTUP_SRCS))
OBJS         := $(SRC_OBJS) $(STARTUP_OBJS)
DEPS         := $(OBJS:.o=.d)

# Two compile rules — one per source directory
$(BUILD_DIR)/%.o: $(SRC_DIR)/%.c | $(BUILD_DIR)
→ $(CC) $(CFLAGS) -c $< -o $@

$(BUILD_DIR)/%.o: $(STARTUP_DIR)/%.c | $(BUILD_DIR)
→ $(CC) $(CFLAGS) -c $< -o $@
⚠ The "missing separator" error explained

When patsubst $(SRC_DIR)/%.c saw startup/startupstm32f4.c it left it unchanged in OBJS. That filename landed in DEPS. -include $(DEPS) tried to include the .c file as a Makefile → line 1 has no tab → "missing separator".

Section 12
12Common Errors & Fixes

Every error you will hit in this project is listed here with its exact cause and fix.

E
missing separator. Stop. Makefile
Cause: patsubst $(SRC_DIR)/%.c doesn't match startup/*.c, leaving the raw .c in DEPS. -include then tries to parse it as a Makefile.
  • Fix: Separate SRC_OBJS and STARTUP_OBJS with separate patsubst calls.
  • Add second compile rule: $(BUILD_DIR)/%.o: $(STARTUP_DIR)/%.c
E
-Wmissing-attributes: specifies less restrictive attribute GCC
Cause: Default_Handler is noreturn but its weak alias declarations are not. GCC requires aliases to be at least as restrictive as their target.
  • Fix: Add noreturn to every handler alias.
  • Use WEAK_HANDLER(name) macro bundling weak + alias + noreturn.
E
HardFault immediately on startup RM0090 §7.3.10
Cause: GPIO register accessed before enabling RCC clock. BusFault → HardFault.
  • Fix: Always write RCC_AHB1ENR |= (1 << 0) before accessing any GPIOx register.
  • Add dummy read of RCC_AHB1ENR after enabling to flush write pipeline.
E
LED glitch at startup RM0090 §8.4.7
Cause: MODER written before BSRR. When MODER goes 00→01, pin briefly drives ODR=0 even if you want HIGH.
  • Fix: Write BSRR first (pre-set latch), then MODER last.
E
BSS zero-fill loop runs forever startup bug
Cause: size-- instead of size1-- in BSS loop — counter never decrements → infinite loop before main().
  • Fix: Use index-based loops (bss[i] = 0) to make this impossible.
E
uint32_t undeclared C error
Cause: <stdint.h> not included. With -ffreestanding, no standard headers are assumed.
  • Fix: Add #include <stdint.h> and #include <stddef.h> to the main header.
E
Vector table not at 0x0800_0000 linker
Cause: Wrong section name — must be ".isr_vector" to match the linker script.
  • Fix: Use __attribute__((section(".isr_vector"))) on the vector table array.
  • Add KEEP(*(.isr_vector)) so --gc-sections doesn't remove it.
